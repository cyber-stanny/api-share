# API Share - 大模型 API 中转平台

## 一句话描述

做一个大模型 API 中转平台，让学生通过学号注册领取自己的 Key，统一调用大模型，平台负责限额和管理。

---

## 用户流程

### 学生端
1. 学生用学号注册（需在合法学号白名单内）
2. 注册后设置登录密码
3. 登录后获得专属 API Key
4. 用该 Key 调用大模型（OpenAI 兼容接口），接入 opencode / claudecode
5. 可以看到自己每天的用量以及剩余额度

### 管理员端
1. 登录管理后台
2. 查看每个学生的用量统计
3. 调整每个学生的每日/每周额度
4. 添加新用户账号
5. 管理学号白名单（增删）

---

## 支持的模型

学生可自由选择模型（可配置）：

| 模型 | 上游来源 |
|------|----------|
| GLM-5.1 | 硅基流动 |
| Mimo-V2.5-Pro | mimo token plan |
| Kimi-2.6 | 硅基流动 |

---

## 控制机制

### 额度控制
- 每个学生有每日上限 + 每周上限（管理员可手动调整）
- 额度用完直接拒绝，返回明确提示

### 限流
- 限制每个学生的请求频率（防刷接口）
- 限制同时并发请求数量（防挤爆）
- 上游频率限制：先 TODO，后续遇到再调整

### 兜底
- 硅基流动余额耗尽 → 拒绝请求，返回提示
- mimo token plan 耗尽 → 拒绝请求，返回提示

## 调用日志

记录每次调用：学生、模型、token 用量、时间、状态码
用途：初期观察使用情况，排查问题
